# Типизированные границы

## Назначение

Типизация защищает границы между UI, application, logic, platform и внешними данными. Она не должна создавать отдельную абстракцию для каждого объекта.

## Правило Protocol

Новый Protocol создаётся только в одном из следующих случаев:

1. существует несколько реальных реализаций;
2. зависимость меняется на внешней системной границе;
3. контракт используется несколькими модулями;
4. граница не позволяет передать целое окно, глобальный runtime или другой слишком широкий объект.

Если внутри одного feature существует одна известная реализация, используется конкретный класс или обычный `Callable`.

Текущий проверенный верхний предел определён в `tests/architecture/test_protocol_budget.py`. Его превышение требует отдельного архитектурного решения, а не механического добавления интерфейса.

## Plugin Store

Внешний JSON преобразуется в проверенные модели logic-слоя.

Application composition создаёт конкретные repository session, operation state и контроллеры fetch, install и uninstall.

UI получает только нужные зависимости и не создаёт application controller самостоятельно.

Одноразовые Protocol для единственной конкретной реализации не применяются.

## Фоновые задачи

`UiTaskRunner` сохраняет тип результата worker и возвращает завершение через UI dispatcher.

Callable запуска worker описан обычным типом функции; отдельный Protocol не нужен, когда существует только одна форма вызова.

## Tk-граница

Узкие Protocol сохраняются там, где код работает с разными Tk-окнами, scheduler host или внешней UI-границей без передачи полного объекта окна.

Динамические параметры Tkinter могут использовать `Any` только на непосредственной внешней границе.

## Проверки

Дизайн защищают две независимые проверки:

1. `scripts/quality/check_typed_boundaries.py` запускает строгую Mypy-проверку выбранных архитектурных границ.
2. `tests/architecture/test_protocol_budget.py` не позволяет бесконтрольно увеличивать количество Protocol-классов.

Эти проверки явно запускаются контуром качества для разработки.
