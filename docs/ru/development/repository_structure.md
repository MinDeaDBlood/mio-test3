# Структура репозитория

Корневой каталог содержит точки запуска, настройки сборки, зависимости и runtime ресурсы.

## Код программы

`src` содержит только код работающей программы.

Внутри `src` не размещаются тесты, ручные проверки, генераторы отчётов и служебные скрипты.

Слои:

1. `src/ui` содержит окна, виджеты, стили, ресурсы интерфейса и presentation логику.
2. `src/app` содержит композицию, application controller и координацию сценариев.
3. `src/logic` содержит прикладные операции и правила работы с проектами, образами и плагинами.
4. `src/core` содержит низкоуровневые алгоритмы форматов и файловых систем.
5. `src/platform` содержит общие адаптеры окружения, процессов, хранения, сети и перезапуска.

## Тесты

`tests` содержит автоматические проверки, распределённые по назначению.

1. `unit` проверяет отдельные функции и классы.
2. `functional` проверяет законченные возможности.
3. `integration` проверяет совместную работу компонентов.
4. `contract` фиксирует публичные интерфейсы и структуру проекта.
5. `regression` защищает исправленные ошибки.
6. `architecture` контролирует границы слоёв и размещение файлов.
7. `smoke` содержит быстрые проверки запуска.
8. `e2e` содержит полные пользовательские сценарии.
9. `release` проверяет сборку и состав архивов.
10. `external` проверяет платформенные зависимости и внешние инструменты.
11. `embedded` содержит проверки окружения, вынесенные из runtime кода.
12. `support` содержит общий тестовый код.

Каждый файл `test_*.py` можно запустить напрямую.

```bash
python tests/unit/core/test_byte_size.py
```

## Служебные скрипты

Все служебные скрипты находятся только в корневом каталоге `scripts`.

1. `scripts/manual` содержит готовые ручные наборы.
2. `scripts/quality` содержит статические и структурные проверки.
3. `scripts/arch_guard` содержит Architecture Guard.
4. `scripts/release` содержит создание и проверку архивов.
5. `scripts/audits` содержит генераторы технических отчётов.
6. `scripts/support` содержит общий код для других скриптов.
7. `scripts/config` содержит конфигурацию Pytest и Mypy.

Файлы `.cmd` в корне `scripts` предназначены для запуска проверок двойным кликом в Windows.

Подробности находятся в [документе о тестах и скриптах](tests_and_scripts.md).

## Документация

1. `docs/en/architecture` и `docs/ru/architecture` содержат зеркальные английские и русские архитектурные документы.
2. `docs/en/development` и `docs/ru/development` содержат зеркальные инструкции для разработчика.
3. `docs/archive/audits` содержит неактивные материалы аудитов, которые не определяют текущую реализацию.
4. `docs/archive/readmes` содержит неактивные переводы README.
5. `docs/README.md` является общим указателем языковых версий.

## Runtime данные

1. `bin` содержит внешние инструменты.
2. `config` содержит настройки программы.
3. `languages` содержит динамически обнаруживаемые языковые JSON файлы.
4. `plugins` содержит базу Plugin Store и установленные плагины.
5. `temp` содержит временные данные.
6. `templates` содержит runtime шаблоны.
7. `logs` содержит журналы программы.

Ресурсы интерфейса находятся в `src/ui/assets`. Корневой каталог `assets` не используется.

## Сборка

`build.py` является единственной корневой точкой сборки программы.

GitHub Actions вызывает только `build.py`. Файлы из `scripts` при сборке не запускаются.

В пользовательский архив не входят:

1. `docs`
2. `tests`
3. `scripts`
4. `src`
5. `.github`
6. Корневой `README.md`
7. Конфигурация инструментов разработки
